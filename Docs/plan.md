# 映像からのモーションキャプチャ実験 プロジェクト計画

## 1. 目的

このプロジェクトは、Pythonを使用してビデオファイルから人物の動きを抽出し、モーションキャプチャデータとして記録・可視化する実験を行うことを目的とします。

## 2. 使用技術

実験の効率性と拡張性を考慮し、以下のライブラリを主要な技術として採用します。

- **OpenCV (`opencv-python`)**: 映像の入出力、フレーム単位での画像処理、結果の描画など、映像に関する基本的な操作を担当します。
- **MediaPipe**: Google提供の機械学習ソリューションライブラリ。高精度かつ高速な姿勢推定（Pose Estimation）機能を利用し、映像内の人物の関節点（ランドマーク）を検出します。

## 3. プロジェクト構成

ソースコード、データ、ドキュメントを分離し、管理しやすい構成を目指します。

```
.
├── Docs/
│   └── plan.md         # このドキュメント
├── data/
│   ├── input/          # 入力ビデオを格納するディレクトリ
│   └── output/         # 出力結果（映像やデータ）を格納するディレクトリ
├── src/
│   └── capture.py      # モーションキャプチャ処理のメインスクリプト
└── requirements.txt    # プロジェクトの依存ライブラリ
```

## 4. 基本処理フロー

モーションキャプチャは以下の流れで実行されます。

1.  **映像読み込み**: `data/input/` から指定されたビデオファイルをOpenCVで読み込む。
2.  **フレームごと処理**:
    - 映像からフレームを1枚ずつ取り出す。
    - MediaPipeの姿勢推定モデルにフレームを入力し、人物の関節点（33点）の座標を取得する。
3.  **結果の可視化**:
    - 取得した関節点と、それらをつなぐ骨格を元のフレーム上に描画する。
4.  **出力**:
    - 描画済みのフレームを新しいビデオファイルとして `data/output/` に保存する。
    - (オプション) 関節点の座標データをCSVやJSON形式でファイルに保存する。

## 5. 開発ステップ

本計画に基づき、以下のステップで開発を進めます。

1.  **環境構築**: `requirements.txt` を作成し、必要なライブラリをインストールする。
2.  **基本実装**: `src/capture.py` に上記の基本処理フローを実装する。
3.  **テスト**: サンプルビデオを用いて、実装した機能が正しく動作することを確認する。
4.  **ドキュメント整備**: `README.md` を作成し、プロジェクトの概要や実行方法を記述する。
